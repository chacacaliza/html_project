## Minggu 1 <br />
a. Day 1 : Unix Command Line <br />
- Terminal basic <br />
  Terminal basic pada windows dikenal dengan sebutan powershell dan commandpromp (CMD)
- Shell merupakan sebutan untuk perantara yang digunakan user dan sistem operasi dalam berkomunikasi <br />
- Command Line merupakan sebutan untuk shell yang berbasis teks <br /> 
- man COMMAND_NAME merupakan command untuk melihat dokumentasi pada sebuah perintah <br /> 
- pwd digunakan untuk melihat posisi directory terkini <br /> 
- cd merupakan command untuk berpindah directory <br /> 
- ls digunakan untuk melihat isi files suatu directory <br /> 
  ls - la untuk melihat isi files yang dihidden <br /> 
- Touch digunakan untuk membuat file pada terminal basic di linux sedangkan "ni" digunakan untuk membuat file di windows <br /> 
- mkdir merupakan command untuk membuat directory baru <br /> 
- cp digunakan untuk menyalin file dan cp -r digunakan untuk menyalin directory <br /> 
- mv digunakan untuk memindahkan atau mengubah nama file dan directory <br /> 
- rm merupakan command untuk menghapus files dan rm -r digunakan untuk menghapus directory <br /> 
  

b. Day 2: Git dan Github <br /> 
- Git merupakan tools dalam memversikan program  <br />
  Git digunakan sebagai version control system serta dapat digunakan untuk melacak segala perubahan pada suatu file atau folder maupun source code <br /> 
- Vendor Git yaitu Github, Gitlab, Bitbucket <br />
  Github merupakan vendor git yang paling umum digunakan dan sebagai tempat penyimpanan Git Repository <br /> 
- Untuk melakukan konfigurasi git :  <br />
  git config global user.name "chaca" <br />
  git config global user.email chaca.caliza@gmail.com <br />
- Untuk melihat hasil konfigurasi : git config --list <br />
- Membuat Repository : git init (dilakukan didalam folder yang dibuat) <br />
  Git add untuk menambah file baru pada Git <br/>
  Git Status digunakan untuk melihat apakah terjadi perubahan atai tidak pada Git <br />
- git remote menghubungkan remote repository dengan project local yang telah kita buat direktorinya <br />
- git commit -m "text commit" digunakan untuk menyimpan perubahan pada Git <br />
- git push -u origin master digunakan untuk mengirimkan perubahan file ke remote repository <br />
